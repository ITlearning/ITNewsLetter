# 웹 아카이브 + 발송 시점 번역/요약 설계

- 작성일: 2026-03-10
- 상태: 사용자 승인 완료

## 1. 목표
- 이미 Discord로 보낸 뉴스를 한 곳에서 다시 볼 수 있는 웹 아카이브를 제공한다.
- 영어 뉴스는 발송 시점에 번역/요약을 생성하고, 웹은 그 결과를 그대로 재사용한다.
- GitHub Pages 기반 정적 배포로 운영 복잡도를 낮춘다.

## 2. 확정 범위
- 웹 데이터 범위: `이미 발송된 뉴스 아카이브`
- 웹 기능:
  - 최신순 목록
  - 소스 필터
  - 슬롯 필터
  - 제목 검색
  - 번역 제목/짧은 요약 표시
  - 원문 링크
- 번역/요약 정책:
  - 영어 기사만 대상
  - 최종 발송된 기사에만 API 호출
  - 기존 `translated_title`, `short_summary` 재사용
  - 모델은 OpenAI만 사용

## 3. 아키텍처
- 원본 데이터:
  - `data/news.json`
- 웹 소스:
  - `site/` 정적 HTML/CSS/JS
- 배포 빌드:
  - `scripts/build_archive_site.py`
- 배포:
  - GitHub Pages Actions workflow

## 4. 데이터 흐름
1. `fetch_and_send.py`가 최종 발송 기사에 번역/요약을 저장한다.
2. `data/news.json`에 발송 아카이브가 누적된다.
3. Pages 배포 시 `build_archive_site.py`가 `news.json`을 읽어 웹용 JSON을 만든다.
4. 웹은 정적 JSON을 fetch해 렌더링한다.

## 5. 중요한 결정
- 웹은 `전체 수집 후보`가 아니라 `발송 아카이브`만 보여준다.
- 번역/요약은 웹 전용 별도 파이프라인을 만들지 않는다.
- 기존 아카이브 항목에 taxonomy 메타데이터가 없어도 Pages 빌드에서 보강한다.

## 6. UI 방향
- 에디토리얼 아카이브 톤
- 큰 타이포 + 가벼운 카드형 목록
- 모바일/데스크톱 모두 읽기 쉬운 구조
- 과도한 대시보드 느낌 대신 기록물/노트 느낌 유지

## 7. 확장성
- 이후 확장 가능:
  - 소스별 페이지
  - 슬롯별 페이지
  - 상세 페이지
  - 유사 기사 묶기
  - 무료/저비용 LLM fallback
- 이번 범위에는 포함하지 않음

## 8. 검증
- 정적 사이트 빌드 스크립트 실행
- 웹 JSON 생성 확인
- 검색/필터 렌더링 확인
- GitHub Pages workflow 문법 확인
